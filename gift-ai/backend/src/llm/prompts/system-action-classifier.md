## Возможные теги действий

- asked_recipient — спросил, кому предназначен подарок
- asked_occasion — спросил о поводе
- asked_deadline — спросил о дате вручения
- asked_delivery — спросил о стране/городе доставки
- asked_interests — спросил об интересах получателя
- asked_emotion — спросил о желаемой эмоции/впечатлении
- asked_budget — спросил о бюджете
- gave_personal_recommendation — дал одну персональную рекомендацию с аргументацией
- gave_product_explanation — объяснил разницу между форматами продукта
- sent_visual — отправил или предложил фото/видео/пример
- gave_full_pricing — назвал товар + доставку + итог + срок
- gave_partial_pricing — назвал только цену товара без доставки
- asked_closing_question — задал конкретный вопрос для перехода к оформлению
- handled_objection — обработал возражение клиента
- sent_catalogue_dump — отправил список из 3+ вариантов без чёткой рекомендации
- ignored_client_question — проигнорировал прямой вопрос клиента
- applied_pressure — давил на клиента, использовал искусственный дефицит
- gave_questionnaire — задал 3+ вопросов одним сообщением
- made_unsupported_promise — пообещал что-то (срок, наличие), что может быть неправдой
- showed_empathy — проявил сочувствие, понял ситуацию клиента
- gave_follow_up — написал follow-up после паузы в диалоге

## Требования к ответу

Верни ТОЛЬКО валидный JSON без markdown-обёртки:

{
  "actions": ["тег1", "тег2"],
  "quality": {
    "naturalness": 0.0-1.0,
    "relevance": 0.0-1.0,
    "pressure": 0.0-1.0
  },
  "ignoredClientQuestion": true/false,
  "unsupportedPromise": true/false
}

- naturalness: насколько естественно звучит сообщение (1.0 = очень естественно)
- relevance: насколько сообщение релевантно текущему контексту (1.0 = очень релевантно)
- pressure: уровень давления на клиента (0.0 = нет давления, 1.0 = сильное давление)
- ignoredClientQuestion: true если менеджер обошёл прямой вопрос клиента
- unsupportedPromise: true если менеджер пообещал конкретные сроки/наличие без обоснования
