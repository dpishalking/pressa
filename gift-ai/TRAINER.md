# Retro Pressa — Trainer Bot

Telegram-тренажёр для менеджеров по продажам. Позволяет отрабатывать навыки продаж на ролевых диалогах с AI-клиентом.

## Быстрый старт

### Через Docker Compose (рекомендуется)

```bash
# 1. Скопируй .env.example → .env
cp .env.example .env

# 2. Заполни обязательные переменные:
# GEMINI_API_KEY — ключ Google Gemini API
# TRAINER_BOT_TOKEN — токен бота (создать через @BotFather)
# ADMIN_API_KEY — произвольный секретный ключ
# GIFT_BOT_TOKEN — токен основного бота

# 3. Запусти все сервисы
cd /path/to/pressa
docker compose up -d

# 4. Проверь статус
docker compose ps
docker compose logs trainer-bot -f
```

### Локальная разработка

```bash
# Backend (включает API тренажёра)
cd gift-ai
npm install
npm run dev                    # http://localhost:3100

# Trainer bot
npm run dev:trainer            # В отдельном терминале
```

```

    │   ├── scenario-loader.ts — Загрузка сценариев из JSON
    │   └── training-service.ts — Бизнес-логика
    └── llm/
        ├── base.ts            — Интерфейс LLMProvider
        ├── gemini-provider.ts — Реализация через Gemini
        └── prompts/           — Системные промпты (6 файлов)

    ├── basic/    (s01–s05)   — 5 базовых сценариев
    ├── medium/   (s06–s13, s31) — 9 средних сценариев
```

## API тренажёра

Все роуты: `http://localhost:3100/trainer/`

| Метод | Путь | Описание |
|---|---|---|
| POST | `/users/register` | Регистрация пользователя |
| GET | `/scenarios` | Список сценариев |
| GET | `/scenarios/:id` | Сценарий по ID |
| POST | `/sessions/start` | Начать тренировку |
| POST | `/sessions/:id/message` | Отправить сообщение |
| POST | `/sessions/:id/finish` | Завершить тренировку |
| GET | `/sessions/:id/evaluation` | Получить оценку |
| GET | `/users/:id/progress` | Прогресс пользователя |

| GET | `/admin/team-analytics` | Аналитика команды (РОП) |
| POST | `/admin/assignments` | Назначить тренировку |

## Режимы тренировки

### Режим A: Сотрудник — менеджер, AI — клиент

- Менеджер получает первое сообщение клиента
- AI-клиент имеет скрытую карточку (менеджер не видит)
- AI не раскрывает информацию сразу
- Состояние клиента обновляется после каждого сообщения
- По завершении — подробный разбор с баллами

### Режим B: AI — менеджер, Сотрудник — клиент

- Сотрудник получает карточку роли клиента
- AI демонстрирует эталонную продажу
- После диалога — обсуждение, что сработало

## Движок состояния клиента

Состояние обновляется после каждого хода по правилам:

```
asked_recipient → emotionalFit +10, trust +5
gave_personal_recommendation → clarity +15, trust +10, choiceOverload -15
sent_visual → clarity +10, trust +8
gave_full_pricing → clarity +15, trust +8, priceAcceptance +8, readinessToBuy +10
sent_catalogue_dump → choiceOverload +20, clarity -10, readinessToBuy -10
ignored_client_question → irritation +15, trust -10
applied_pressure → irritation +15, trust -20
```

Клиент готов к покупке при:
- trust ≥ 60, clarity ≥ 70, emotionalFit ≥ 60
- readinessToBuy ≥ 70, irritation < 50

## Добавление нового сценария

1. Создай файл `gift-ai/backend/scenarios/<difficulty>/s<N>-name.json`
2. Следуй структуре существующих сценариев
3. Перезапусти backend — сценарий загрузится автоматически
4. Или добавь через API `/admin/scenarios` (если требуется)

## Дашборд РОПа

Откройте `http://localhost:8080/training` для:
- Heatmap навыков команды (зелёный/жёлтый/красный)
- Рейтинг сотрудников
- Слабые навыки команды
- Зона риска (сотрудники с низким баллом)
- Частые ошибки
- Назначение тренировок через API
